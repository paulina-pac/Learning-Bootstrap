Learning Bootstrap

Few things worth mentioning about bootsrap

  • It uses REBOOT.CSS styles, its job is to normalize styles so that they look similar across devices and browsers
  • It uses REMS, 1REM = 16px
  • The default margin-top is set to 0, bootstrap only adds margin at the bottom of the elements
  • It uses inherit property whenever possible
  • It uses border-box instead of content-box
  • It uses native font stacks, so it tries to use whatever the current platoform defines as the default SANS-SERIF

In Bootsrap if you give an element class like h3, its style will look like tag h3

If you want to use something dramatic, class = display-1 to display-4, is a way to go.

class = "lead" on the other hand will make the text lighter and spaced

Bootsrap typographic utilities

So bootstrap uses everything as css does eg

  • text-justify, which lets the text of a paragraph or any other element, fit to the left and the right with the same amound of spacing.
  • text-nowrap, allows us to run text infinetely to one side withou actually cutting it off at the edge, especially when you want to take one line and NEVER WRAP.

  • Text-Styles, eg. font-weight-bold, font-weight-normal and font-italic

Mastering Bootstrap Layout

As you may (or may not) know, the most useful part of Bootstrap is the grid, so this is the frist thing you should master!

Bootstrap layouts have 3 main pieces. The first is containers, which take up 100% of the width of the view port. Inside containers, you can also place content in rows & columns. Bootsrap uses 12 column grid system, that has breakpoints for: extra-small, small, medium, large and x-large devices.

Containers & Rows

The Grid fun facts

  • It's a responsive 12-column system
  • It uses flexbox, for the easier creation of complex layouts
  • So make sure you master 3 simple components: Containers, used with or without the grid to align content to the viewport, or center it around a set of breakpoints. Rows & Columns to allow for the layout creation.

Grid Containers

There are 2 different types of containers. regular ones, which center content and snap to certain grid points, or fluid ones, which are always the full width of the viewport. One of the reason you use a container is because you get a 15px padding on each side to make sure it works well with backgrounds and other elements.

Rows & Columns / col-BP-COL

Column fun facts

  • It uses 12 column grid, and how content fits into columns is up to you, and you can specify when the layout transform into full horizontal one.
  • Columns can span, just type in col(-BP) (-COL)

      • BP:

    • sm < 576px
    • md < 768px
    • lg < 992px
    • xl < 1200px

      • COL:

    • 1 – 12
  • If you just specify class col, Bootstrap will try to fit that column into the layout, so if you have more than one column, it will automatically size it, to fit into that space, so if we have 2 columns, bootstrap will make them 50% size, 3 columns will take 33# of the contener size itd.
  • if we type col-sm the column will take normal width of the browser, untill it reaches that sm point of 576px or higher, if we type col-md that column will continue to take the full width of the space in the viewport/browser untill the medium breakpoint of 768px.
  • The last col porperty is the number of columns it should display in the breakpoint col-md-6

Bootstrap Display Properties

There are other ways beside the grid to control how elements are positioned within Bootstrap, so let's take a look at what they are and how to use them. Now, first of all is position. These work exactly like the regular CSS position property. There are three options you can use. You can make an element fixed to the top of the screen, which means it would get removed from the flow of the document and put essentially at the top of the window or at the top of the view port, fixed-bottom would place the element at the bottom. There's another one called sticky-top, and that one will allow you to have an element stick to the top of the view port or the screen as you scroll in elements. There's also display properties.

The display properties mimic what is possible with CSS and opens up Bootstrap to Flexbox classes that can help with layout. So use a display class, you use the d and then optionally a breakpoint, and then the type of display property that you want. And you can see that we have the full range of display properties that you have normally available in CSS also in Bootstrap. So you can say d and the none or display none at a specific breakpoint. So you would say like, d, then -md, and then -none. That would display nothing at the medium breakpoint. So these are actually quite powerful. Bootstrap gives you a great way to control how your display properties are working at different breakpoints. Now, Flex has a ton of options.
