Learning Bootstrap

Few things worth mentioning about bootsrap

  • It uses REBOOT.CSS styles, its job is to normalize styles so that they look similar across devices and browsers
  • It uses REMS, 1REM = 16px
  • The default margin-top is set to 0, bootstrap only adds margin at the bottom of the elements
  • It uses inherit property whenever possible
  • It uses border-box instead of content-box
  • It uses native font stacks, so it tries to use whatever the current platoform defines as the default SANS-SERIF

In Bootsrap if you give an element class like h3, its style will look like tag h3

If you want to use something dramatic, class = display-1 to display-4, is a way to go.

class = "lead" on the other hand will make the text lighter and spaced

Mastering Bootstrap Layout

As you may (or may not) know, the most useful part of Bootstrap is the grid, so this is the frist thing you should master!

Bootstrap layouts have 3 main pieces. The first is containers, which take up 100% of the width of the view port. Inside containers, you can also place content in rows & columns. Bootsrap uses 12 column grid system, that has breakpoints for: extra-small, small, medium, large and x-large devices.

Containers & Rows

The Grid fun facts

  • It's a responsive 12-column system
  • It uses flexbox, for the easier creation of complex layouts
  • So make sure you master 3 simple components: Containers, used with or without the grid to align content to the viewport, or center it around a set of breakpoints. Rows & Columns to allow for the layout creation.

Grid Containers

There are 2 different types of containers. regular ones, which center content and snap to certain grid points, or fluid ones, which are always the full width of the viewport. One of the reason you use a container is because you get a 15px padding on each side to make sure it works well with backgrounds and other elements.

Bootstrap Display Properties

There are other ways beside the grid to control how elements are positioned within Bootstrap, so let's take a look at what they are and how to use them. Now, first of all is position. These work exactly like the regular CSS position property. There are three options you can use. You can make an element fixed to the top of the screen, which means it would get removed from the flow of the document and put essentially at the top of the window or at the top of the view port, fixed-bottom would place the element at the bottom. There's another one called sticky-top, and that one will allow you to have an element stick to the top of the view port or the screen as you scroll in elements. There's also display properties.

The display properties mimic what is possible with CSS and opens up Bootstrap to Flexbox classes that can help with layout. So use a display class, you use the d and then optionally a breakpoint, and then the type of display property that you want. And you can see that we have the full range of display properties that you have normally available in CSS also in Bootstrap. So you can say d and the none or display none at a specific breakpoint. So you would say like, d, then -md, and then -none. That would display nothing at the medium breakpoint. So these are actually quite powerful. Bootstrap gives you a great way to control how your display properties are working at different breakpoints. Now, Flex has a ton of options. Bootstrap visibility

As you may or may not know, you can hide and show content on the page, and it's easily done with bootstrap by using visibility class.

invisible, visible class: It doesn't really change the display property itself. Invisible will make an element not appear on screen, but it will take up the amount of room that it would normally take up. Whereas visible means that it will be visible, but only to screen readers. So it's a way to force an element to show up with somebody with assistive devices.

  • invisible
  • visible
The other thing of course would be to simply display-property to none (d-none) You can set display to none, but add class visible, it will be visible for assistive devices. You can also hide elements in specific breakpoints that way. e.g d-sm-none
  • d-none visible

Nav Dropdowns

Dropdowns are used very often in menus. A dropdown requires a container with a class of dropdown, this is usually a div, or it could be on a list item if it's a nav that uses list items. That container is used to align the dropdown to the proper item in the navigation. There's also a dropdown-toggle class, and that is added on the link in order to get a little triangle icon to indicate that this element has a dropdown in it.

Dropdowns can be activated in one of two ways: through JavaScript or by using a data attribute, and this attribute is called data-toggle, and you set it to dropdown. With a dropdown menu class, we need to create another container for that, and then you insert a series of links inside this dropdown menu. Each of the dropdown links gets a dropdown item class, and you should so add an id as well as some screen reader classes to make everything work well for accessibility devices. Now, the id is going to allow you to link the aria attributes with the specific container.
